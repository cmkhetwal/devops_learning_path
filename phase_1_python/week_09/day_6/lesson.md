# Week 9, Day 6: Practice Day - Docker & Containers

## What You'll Practice
Today you combine everything from Week 9 into five mini-projects that
simulate real DevOps Docker automation tasks.

## Mini-Project Overview

### Project 1: Docker CLI Tool
Build a command-line interface that simulates `docker ps`, `docker images`,
and `docker stats` using the mock classes you have built this week.

### Project 2: Container Health Dashboard
Create a dashboard that checks container health, displays stats, and flags
containers that need attention.

### Project 3: Compose Generator
Build a tool that takes user input (service name, image, ports) and generates
a complete docker-compose.yml file.

### Project 4: Image Cleanup Script
Write a script that analyzes local images, identifies ones that can be
removed (old, untagged, oversized), and simulates cleanup.

### Project 5: Deployment Manager
Build a deployment manager that can "deploy" a new version of an application
by stopping old containers and starting new ones (all simulated).

## Tips
- Reuse the mock classes from Days 1-5
- Focus on clean code structure -- functions and classes
- Think about error handling and edge cases
- Print clear, formatted output

## DevOps Connection
These projects mirror real tools that DevOps engineers build and use daily.
The patterns you practice today will transfer directly to production work
with real Docker environments.
